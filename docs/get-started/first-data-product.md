# Build Your First Data Product

This tutorial maps the Customer 360 data-product files to the artifacts Floe and dbt generate before Dagster runs anything.

## Prerequisites

- Repository dependencies installed with the project `uv` environment.
- The demo files present under `demo/customer-360/`.
- No Kubernetes cluster is required for this compilation walkthrough.

## What This Does

Customer 360 uses dbt for SQL ownership and Floe for platform compilation. dbt owns model SQL, dependency parsing, and the generated `target/manifest.json`. Floe reads the data-product spec and platform manifest, validates platform choices, and writes `compiled_artifacts.json`, which is the contract consumed by downstream runtime code. Dagster starts later at the run boundary: it loads compiled artifacts and dbt outputs rather than taking direct ownership of SQL compilation.

The important files are:

- `demo/customer-360/floe.yaml`: the data-product declaration for Customer 360.
- `demo/manifest.yaml`: the platform manifest that selects platform services and plugins.
- `demo/customer-360/dbt_project.yml`: the dbt project configuration.
- `demo/customer-360/models/`: the SQL models and schema metadata owned by dbt.
- `demo/customer-360/target/manifest.json`: the dbt compile output.
- `demo/customer-360/compiled_artifacts.json`: the Floe compile output and runtime contract.

For the artifact schema, see [Compiled Artifacts](../contracts/compiled-artifacts.md).

## Steps

```bash
ls demo/customer-360
make compile-demo
git diff -- demo/customer-360/target/manifest.json demo/customer-360/compiled_artifacts.json
```

## Expected Output

`ls demo/customer-360` should show the source files and generated outputs, including:

```text
dbt_project.yml
floe.yaml
models
target
compiled_artifacts.json
```

`make compile-demo` should compile every demo product and end with:

```text
All demo products compiled successfully!
```

The `git diff` command should be empty when committed artifacts are current. If it prints changes for `target/manifest.json` or `compiled_artifacts.json`, review them as generated artifact updates before committing.

## Troubleshooting

- If dbt compilation fails, inspect `demo/customer-360/models/` first; Floe should not parse or rewrite SQL.
- If `compiled_artifacts.json` changes unexpectedly, compare it with `demo/customer-360/floe.yaml` and `demo/manifest.yaml`.
- If Dagster later cannot find assets, confirm both `target/manifest.json` and `compiled_artifacts.json` were regenerated by `make compile-demo`.
